Transform Rules

Cloudflare Transform Rules

Rewrite URLs and add, change, or remove HTTP headers at the edge — without touching your origin. Three transform types plus one-click Managed Transforms.

URL Rewrite

Rewrite the path/query sent to origin. Invisible to the visitor (unlike a redirect).

Request Header Modification

Add/set/remove headers on the request before it reaches origin.

Response Header Modification

Add/set/remove headers on the response sent back to the visitor.

Configure first: Dashboard → Rules → Transform Rules. Response-header changes are visible to the browser (tested below). Request-header and URL-rewrite changes are only visible at the origin — see them on CF Echo.
Use at your own risk. This lab was built by TPimenta LAB for hands-on testing. Always consult the official Cloudflare documentation for accurate, up-to-date guidance.

Example rules

URL Rewrite — serve /promo from /campaigns/2026/ invisibly:

Type:        Rewrite URL
Expression:  (http.request.uri.path eq "/promo")
Path:        Rewrite to  /campaigns/2026/index.html  (static)

Response Header Modification — add a security header:

Type:        Modify Response Header
Expression:  true
Action:      Set static
Header:      X-Frame-Options = SAMEORIGIN

Request Header Modification — forward a dynamic value to origin:

Type:        Modify Request Header
Expression:  true
Action:      Set dynamic
Header:      X-Client-Country = ip.src.country

Managed Transforms

Toggle-based shortcuts (no expression needed). Dashboard → Rules → Settings.

Managed TransformEffect
Add visitor location headersAdds CF-IPCountry, CF-IPCity, lat/long, etc. to the request to origin.
Add security headersSends HSTS and other hardening headers in responses.
Remove "X-Powered-By"Strips framework fingerprint headers from responses.
Add true client IP headersAdds True-Client-IP for origins that expect it.
