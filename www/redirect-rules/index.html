Redirect Rules

Cloudflare Redirect Rules

Dynamic redirects evaluated at the edge before a request reaches your origin. Define a matching expression and a target with status code, query handling, and even dynamic concat() expressions.

Configure first: Dashboard → Rules → Redirect Rules. The live test below follows whatever rules you've created on this zone. With no rule, paths just hit the origin.
Use at your own risk. This lab was built by TPimenta LAB for hands-on testing. Always consult the official Cloudflare documentation for accurate, up-to-date guidance.

Live redirect test

The browser follows redirects and reports the final URL and whether a redirect occurred. (With redirect: "manual" the Location header is opaque to JS, so we follow and read response.redirected + response.url.)

Run a test to see the result.

Redirect status codes

CodeMeaning
301Moved Permanently — cached by browsers/SEO. Method may change to GET.
302Found (temporary). Method may change to GET. Default for most redirects.
307Temporary Redirect — preserves the original HTTP method and body.
308Permanent Redirect — preserves method/body; permanent variant of 307.

Example rules

Static target, query preservation, and a dynamic target built from the path.

Static redirect (When incoming requests match…):

Expression:  (http.request.uri.path eq "/old-blog")
Target:      https://www.pimenta.fun/blog/
Status:      301
Preserve query string: on

Dynamic redirect with concat() — map /go/<slug> to docs:

Expression:  starts_with(http.request.uri.path, "/go/")
Target:      concat("https://developers.cloudflare.com/",
                     substring(http.request.uri.path, 4))
Status:      302
Expression mode: Dynamic

Redirect Rules vs Bulk Redirects

Use casePick
A handful of pattern-based redirectsRedirect Rules — expression engine, dynamic targets, per-zone.
Thousands of 1:1 URL mappingsBulk Redirects — account-level lists (CSV), great for migrations.
